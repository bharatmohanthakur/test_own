### Key Insight: **CoT distillation is essential**
- Without thinking traces: cipher=0%, gravity=12%
- With Qwen3.5 27B distillation: cipher=75.5%, gravity=98.8%
- **bit_manipulation is the hardest**: even with distillation only 10-19%
- bit_manipulation tokens are very long (need 16K+ for decent accuracy)
- "Poisonous" examples from verbose distillers reduce bit_manipulation accuracy

## ACTIONABLE: What Top Teams Are Doing

1. **Distilling CoT from strong models** (Qwen3.5 27B, DeepSeek-R1) -- the #1 differentiator
2. **Generating synthetic training data** for each puzzle type with verified answers
3. **Controlling output length**: bit_manipulation needs concise solutions (8K token limit hurts)
4. **Programmatic CoT**: Writing out actual computation steps (g = 2d/t^2, letter mapping tables)
5. **75%/25% reasoning split**: Unsloth docs recommend 75% reasoning + 25% non-reasoning data

## GRPO Status: Difficult on Kaggle
- Requires 2x model copies in memory (impossible in bf16 on 48GB)
- Unsloth doesn't support fast_inference for Mamba hybrid models
- vLLM LoRA loading fails for conv1d layers
- One user reports "vLLM + sleep mode worked" but no details
- HuggingFace GRPO trainer does NOT support RAM sharing like Unsloth
