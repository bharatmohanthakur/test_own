## Techniques from Research

### Nemotron-CrossThink (NVIDIA paper)
1. **Multi-domain data blending**: 2:1 ratio of general reasoning to math — 13.36% accuracy gain
2. **Difficulty-based filtering**: Remove easy examples (solvable by smaller models) — +2.15% gain
3. **Question format standardization**: Open-ended format — +1.21% gain
4. **Short-form answer templates**: Outperform long-form by 1.20%
5. **GRPO for RL training on filtered, blended data**

### Nemotron-Cascade 2 (NVIDIA, March 2026)
- Sequential domain-wise RL: SFT -> RLHF -> IF-RL -> Math RL -> Code RL -> SWE RL
- Uses GRPO with strict on-policy training throughout
- Each domain gets tailored hyperparameters and curriculum
- RL is resistant to catastrophic forgetting between domains
- Nemotron-Cascade-14B surpasses DeepSeek-R1-0528 (671B)

### AIMO-2 Winning Solution (NemoSkills)
- 540K unique math problems with 3.2M long-reasoning solutions
- Tool-Integrated Reasoning (TIR): model writes Python code during reasoning
- Generative Solution Selection (GenSelect): train model to pick best solution
- Used DeepSeek-R1 and QwQ-32B to generate synthetic solutions

### Unsloth for Nemotron
- 16-bit LoRA needs ~60GB VRAM (fits on RTX Pro 6000 48GB? Might need quantization)
- Dataset: 75% reasoning + 25% non-reasoning to preserve capabilities
- NEVER fine-tune router layers (disabled by default in Unsloth)
- Unsloth Colab notebook: Nemotron-3-Nano-30B-A3B SFT LoRA on A100
- NeMo Gym Sudoku GRPO notebook available

### GRPO Implementation (Unsloth tutorial)
- Reward functions: correctness match (+1), format compliance (+1), length penalty (-1)
- GRPOConfig: use_vllm for fast inference, num_generations per prompt
- Loss types: grpo, dr_grpo, dapo
- Minimum 300 steps (~30 min), longer = better reasoning chains
- Save LoRA weights with model.save_lora()
