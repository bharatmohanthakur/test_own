### 2. GRPO on Nemotron is broken without a transformers upgrade
- Source: https://www.kaggle.com/competitions/nvidia-nemotron-model-reasoning-challenge/discussion/690161 — "Why GRPO is Painfully Slow on Nemotron (and the Fix)" (Komil Parmar, 16 votes)
- Root cause: `modeling_nemotron_h.py` has a param-name mismatch — `prepare_inputs_for_generation` passes cache as `past_key_values` but `forward()` expects `cache_params`. Cache is silently dropped → 2 tok/s instead of 38 tok/s → 20x slowdown during generation.
- **Fix**: `transformers >= 5.3.0` has native NemotronH support with the bug fixed.
  - `pip install --no-deps --force-reinstall transformers-5.5.3` (force needed because Kaggle preinstalls an older one)
  - **Drop `trust_remote_code=True`** — otherwise it pulls the old broken `modeling_nemotron_h.py` from the model repo and silently overrides the fix
  - Set `gradient_checkpointing=False` — the native impl doesn't declare support for it and will raise ValueError
- Numbers: 2 tok/s → 38 tok/s. 500-step GRPO run goes from "days" to "fits in a Kaggle session"
- **ACTION**: If doing any GRPO after v22, upgrade transformers first. Relevant to memory error #27.

### 4. Tong Hui Kang's winning "secret" on bit_manipulation: algorithmic CoT
- Source: https://www.kaggle.com/competitions/nvidia-nemotron-model-reasoning-challenge/discussion/690307
- His bit_manipulation algorithm solves **1364/1602 = 85.1%** of training binary problems.
- Brings overall validation from 74.53% → ~85% if trained perfectly. **This one puzzle type alone is worth the full gap to first place.**
- Algorithm core idea: don't brute-force all 10,584 ROT/SHL/SHR × 6-op × 3-transform combos (would blow the 7680 token budget). Instead:
  - Consider 18 unary combos (8 positions + 8 negated + 2 constants)
  - Consider 336 binary combos (8×7 positions × 6 operations)
  - For each pair, iterate output bits and match. Total 354 combos, ~15 tokens each = 5130 tokens. Fits budget.
  - Uses +1/+1 modulo-8 index progression trick (`AND51 → AND62 → AND73`) to match left-to-right.
  - Picks longest matching sequence, tie-break by earliest appearance.
- **Full CoT example** on gist: https://gist.github.com/tonghuikang/6312fb04a0149c9a3e28bd7c2a844e9a (~3500-4000 tokens per sample)
- Sections: Output data tables → Input data tables → Transformation scan (18+336 combos) → Left/right best selection → Tentative mapping → Apply to test input → `\boxed{}`
- Only limitation: fails on `(SHL(x) XOR SHR(y)) AND ROT(z)` when `x+y < 8` (3 relevant input bits per output bit — can't happen with pairs-only match)
- **ACTION**: This is THE biggest lever. Implement his algorithm as a Python generator, produce 1600 synthetic traces, train SFT. Expected delta: +0.10-0.12 on leaderboard. Do this regardless of whether Tong Hui Kang's notebook drops tomorrow — it's the single largest win.
