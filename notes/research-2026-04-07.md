## Key Findings

### 1. Reuse the v22 adapter as a notebook input, not a download
- `kaggle kernels output` did not reliably fetch the large `adapter_model.safetensors`
- Direct notebook reuse via `kernel_sources` is the correct path

### 2. Kaggle notebook outputs can mount under a deeper path
- Working adapter path for v22:
  - `/kaggle/input/notebooks/bharatmohan/nemotron-sft-v22/nemotron-lora-adapter`
- Do not assume notebook outputs mount directly under `/kaggle/input/<slug>/...`
- Adapter search should include deeper glob depths

### 3. The P100 failure is architecture mismatch, not memory
- Relevant log:
  - `Tesla P100-PCIE-16GB with CUDA capability sm_60 is not compatible with the current PyTorch installation`
  - `The current PyTorch install supports CUDA capabilities sm_70 sm_75 sm_80 sm_86 sm_90 sm_100 sm_120`
- This is the standard CLI-push trap: Kaggle starts a P100 run automatically unless the notebook is re-saved in the UI on RTX Pro 6000
- Do not use this failure to judge GRPO VRAM requirements
- Version-3 log dies during `FastLanguageModel.from_pretrained(...)` before adapter load. It never reaches:
  - `SFT adapter loaded`
  - `GRPO dataset`
  - `Patched is_fast_path_available = False on model modules`
- Conclusion: the current post-adapter shape-mismatch fix has not been tested yet

### 4. Adapter-loaded GRPO hit a known Mamba fast-path issue
- Failure:
  - `RuntimeError: mat1 and mat2 shapes cannot be multiplied (...x131072 and 2688x131072)`
- Location:
  - inside Unsloth/GRPO generation-scoring path after `PeftModel.from_pretrained(...)`
- Practical fix:
  - patch `is_fast_path_available = False` on the actual `model.modules()` after adapter load
  - patching only imported `modeling_nemotron_h` modules is not sufficient in this path

### 5. Safer GRPO trainer settings for this path
- Keep these explicit in `GRPOConfig`:
  - `bf16=True`
  - `gradient_checkpointing=True`
  - `gradient_checkpointing_kwargs={"use_reentrant": False}`
  - `use_vllm=False`
  - `remove_unused_columns=False`
  - `loss_type="grpo"`
