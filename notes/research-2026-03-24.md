## Techniques from Research

### 1. Data Generation (BIGGEST LEVER)
- **Programmatic puzzle generation**: Write Python solvers for each of the 6 puzzle types, generate 10x-100x more training examples with verified answers
- **Distillation from larger models**: Use DeepSeek-R1, Qwen3, or GPT-4 to generate high-quality CoT for training problems
- **Curriculum learning**: Sort samples by difficulty, train easy first (NVIDIA blog says this "significantly improved stability and final performance")
- **75/25 reasoning mix**: Unsloth recommends 75% reasoning (with <think>) + 25% direct answers to preserve model capabilities

### 2. LoRA Configuration Best Practices
- **Recommended target modules**: q_proj, k_proj, v_proj, o_proj, gate_proj, up_proj, down_proj, in_proj, out_proj
- **NEVER target router layers** in MoE — will collapse expert specialization
- **Rank**: 16-32 (competition max is 32). NVIDIA blog found rank 64 optimal but we're capped at 32
- **Alpha**: rank or 2*rank (32 or 64 for rank 32)
- **Dropout**: 0 (Unsloth recommends)
- **VRAM**: ~60GB for 16-bit LoRA fine-tuning
- **Context**: Validate samples against 2048 tokens (but eval uses 8192 max)

### 3. Training Recipe (NVIDIA Blog)
- **SFT is sufficient** for big gains — NVIDIA blog shows "10-point improvements over base models on GPQA/MMLU"
- **Consistent reasoning emerged after ~13 hours of training**
- **LR**: 0.0001 with cosine warmup (5%)
- **Weight decay**: 0.001
- **Batch size**: 256 effective (with gradient accumulation)
- **Steps**: 2,000+ minimum
- **Data**: ~500,000 balanced samples recommended

### 4. GRPO vs SFT vs MOPD
- **SFT**: Good baseline, 10+ point gains possible with good data
- **GRPO**: NVIDIA uses it in cascade RL, on-policy with no KL
- **MOPD (Multi-domain On-Policy Distillation)**: New technique from Nemotron-Cascade 2 (March 16, 2026). More sample-efficient than GRPO. Dense token-level signal vs sparse sequence-level reward
- **Practical takeaway**: SFT with high-quality data is likely enough to close the gap. GRPO may help for the last 2-5%

### 5. NeMo Skills Pipeline
- NVIDIA open-sourced NeMo-Skills for synthetic data generation
- Supports math, code, science benchmarks
- NeMo Data Designer for generating synthetic data from seed data
- Can be used to generate puzzle-specific training data

Then use these to:

- numeral_system is ~100% zero-shot, so skip it in training

### 3. SCALE TRAINING DATA
- Current: 9,500 examples (2,000 used in v5 SFT)
